- add actual serverless/function routes supported by the deployment platform for:

  - /api/sage/query

OR

B. If the platform expects a server handler / functions directory:
- restructure Sentrix to use that supported serverless pattern
- wire the frontend to those same-origin function endpoints

Requirements:
- no separate app
- no separate product
- no second domain
- same-origin API only
- production routes must return JSON, never HTML
