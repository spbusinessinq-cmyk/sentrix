Add Investigation Mode to Sentrix as a lightweight intelligence workflow layered on top of the existing search, inspect, save, bookmark, and collections systems.

IMPORTANT:
Do not redesign the UI from scratch.
Do not break current search, inspect, save, bookmark, collection, or URL-routing behavior.
Do not add fake telemetry or fake AI claims.
This pass is about turning Sentrix into a usable research workflow, not a dashboard.

MISSION:
Make Sentrix feel like an intelligence workspace where a user can:
- search
- inspect
- save
- collect
- build a focused investigation session
- review/export what they found

PRIMARY GOALS:
1. add a real Investigation Mode
2. make saved items attach to the current investigation
3. make the investigation visible and useful
4. keep everything lightweight, fast, and credible

━━━━━━━━━━━━━━━━━━━━━━━
1. INVESTIGATION MODE TOGGLE
━━━━━━━━━━━━━━━━━━━━━━━

Add a clear Investigation Mode toggle in the Sentrix UI.

Behavior:
- OFF = normal search workflow
- ON = search/inspect/save activity attaches to the active investigation

When enabled:
- create a current investigation automatically if none exists
- default name can derive from the first search query
- allow renaming later

Keep the toggle subtle and premium, not noisy.

━━━━━━━━━━━━━━━━━━━━━━━
2. CURRENT INVESTIGATION STATE
━━━━━━━━━━━━━━━━━━━━━━━

Create a lightweight investigation state model persisted locally.

Each investigation should store:
- id
- name
- createdAt
- updatedAt
- query seed / starting query
- saved items
- collected items
- bookmarked items tied to the investigation
- notes (simple text)
- source counts
- posture counts

Persist it cleanly in localStorage.

━━━━━━━━━━━━━━━━━━━━━━━
3. AUTO-ATTACH BEHAVIOR
━━━━━━━━━━━━━━━━━━━━━━━

When Investigation Mode is ON:

- Inspect on a result should optionally attach that item to the current investigation
- Save should attach it
- Add to Collection should attach it
- Bookmark can optionally attach or remain separate, but the relationship should be clear

Keep this friction-light:
- avoid too many confirmation dialogs
- prefer sensible default auto-attach behavior with visible feedback

━━━━━━━━━━━━━━━━━━━━━━━

- investigation name
