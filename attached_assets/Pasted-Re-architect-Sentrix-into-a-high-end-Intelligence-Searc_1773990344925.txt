Re-architect Sentrix into a high-end Intelligence Search Engine.

IMPORTANT:
This is a product identity pivot, not a cosmetic tweak.
Sentrix is no longer a browser-first product.
Do not frame it as a full browser.
Do not rely on internal website rendering as the primary experience.
External opening of links is expected and correct.

MISSION:
Transform Sentrix into a serious intelligence search environment where users:
- search the web
- evaluate source posture before clicking
- inspect pages and domains
- save findings
- build collections
- move with clarity and control

Sentrix should feel like:
- a premium intelligence search system
- a research command surface
- a pre-click analysis engine
- a disciplined alternative to generic search

NOT:
- a browser clone
- a dashboard
- a toy “threat feed”
- a fake cyber UI

1. PRODUCT IDENTITY — HARD PIVOT
━━━━━━━━━━━━━━━━━━━━━━━

Sentrix is now:

“SENTRIX — Intelligence Search Engine”
“Search clearly. Decide before you click.”

BLACKDOG remains:
- hidden engine
- status only
- no fake detections
- no user-facing engine controls

Make the entire product feel intentionally built around search, evaluation, and external decision-making.

De-emphasize the old browser framing:
- website rendering is secondary
- search and inspection are primary
- external opening is a normal action

━━━━━━━━━━━━━━━━━━━━━━━

Build a strong premium landing/search surface with:

CENTERPIECE:
- large SENTRIX brand

  “Search clearly. Decide before you click.”

  “Intelligence-powered search with BLACKDOG analysis.”

PRIMARY UI:

SECONDARY SECTIONS:
A. Quick Actions

- BLACKDOG engine status

C. Recent / Saved Surface
- recent searches
- recent inspected links
- saved/bookmarked items

The page must feel useful the second it opens.

3. SEARCH RESULTS — MAKE THEM THE PRODUCT
━━━━━━━━━━━━━━━━━━━━━━━

Search Results must become the main experience.

Each result card should feel like an intelligence object, not a generic blue link.

Every result should include:

CORE:
- title
- domain
- snippet
- source type
- risk posture
- reasoning line

SOURCE TYPE OPTIONS:
- News
- Documentation
- Reference
- Commercial
- Forum
- Download
- Unknown

RISK POSTURE:
- SAFE
- CAUTION
- UNKNOWN
- DANGER

REASONING EXAMPLES:
- Common reference domain
- Commercial destination
- Login/verification language detected
- Download-oriented result
- Insecure protocol
- Unverified source posture

ACTIONS:
- Open externally
- Inspect
- Save
- Add to Collection

Do not show fake tracker/script counts.
Do not invent telemetry.
Keep the analysis simple, believable, and useful.

4. INTELLIGENCE ENRICHMENT LAYER

Build a real first-pass enrichment system over search results and URLs.

Use:
- URL pattern logic
- protocol detection
- keyword heuristics
- domain categorization
- result metadata if available from the search provider

Generate:
- source type
- posture/risk label
- short reasoning line
- simple notes for inspect view

Signals can include:
- http vs https
- presence of “login”, “verify”, “reset-password”
- presence of “download”, “free”, “setup”, “installer”
- known reference domains
- known documentation domains
- file/download-type patterns

Keep it disciplined and transparent.
No fake advanced claims.
